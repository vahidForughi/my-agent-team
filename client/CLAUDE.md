# client — legacy Angular SPA

## What & why

The original monolithic Angular web client covering product browsing, cart, checkout, and account
flows. It is **legacy** — being superseded by the `micro-frontends/` React suite. Treat it as
maintenance-only; confirm with the user before investing in new features here.

## Where it lives

`client/` — Angular CLI application (`angular.json` declares a single `client` project).

- `client/src/app/` — feature modules: `home/`, `store/`, `basket/`, `checkout/`, `account/`,
  plus `core/` (navbar, footer, header, guards, interceptors, services) and `shared/` (models,
  order-summary, shared services).
- `client/src/environments/` — `environment.ts` (dev) and `environment.prod.ts` (prod template,
  generated at build time).
- `client/angular.json` — build/serve/test/lint architect targets, prod budgets, file replacements.
- `client/generate-env.cjs` — injects deployment env vars into `environment.prod.ts`.

## Tech stack

From `client/package.json`:

- **Framework:** Angular 21 (`@angular/core` `^21.2.16`), TypeScript `~5.5.2`, RxJS `~7.8.2`,
  zone.js `~0.16.2`. ESM package (`"type": "module"`).
- **UI:** Bootstrap 5 (`bootstrap` `^5.3.6`), `ngx-bootstrap` `^21.2.0`, Font Awesome,
  `ngx-spinner`, `ngx-toastr`, `xng-breadcrumb`, AOS. Styles are **SCSS** (`angular.json`
  `inlineStyleLanguage: scss`, component schematic `style: scss`) — note this differs from
  `micro-frontends/` which uses LESS.
- **Auth:** Azure MSAL (`@azure/msal-angular` `^3.0.23`, `@azure/msal-browser` `^3.28.1`) plus
  `oidc-client` `^1.11.5`.
- **Build/test toolchain:** `@angular-devkit/build-angular` (browser builder), Karma + Jasmine,
  `angular-eslint` (lint is stubbed — see Configuration/Gotchas).

## Build / run / test

Prerequisite: Node + npm; dev server runs on **port 4200**.

```bash
cd client
npm install
npm start            # ng serve → http://localhost:4200 (development config)
npm run build        # ng build (default config = production)
npm run build:prod   # node generate-env.cjs && ng build --configuration=production
npm run watch        # ng build --watch --configuration development
npm test             # ng test (Karma + Jasmine)
npm run test:ci      # ng test --watch=false --browsers=ChromeHeadless || true
npm run lint         # STUB: echoes a message and exits 0 (see Gotchas)
```

## Configuration

- **Dev config** lives in `client/src/environments/environment.ts` (edited directly).
- **Prod config** is generated by `client/generate-env.cjs`, which restores a placeholder template
  in `environment.prod.ts` and substitutes env vars (idempotent). Variables and their defaults:
  - `API_URL` → `http://localhost:8010` (the gateway)
  - `CLIENT_URL` → `http://localhost:4200`
  - `IDP_AUTHORITY` → `https://localhost:9009`
  - `AZURE_B2C_CLIENT_ID`, `AZURE_B2C_AUTHORITY`, `AZURE_B2C_KNOWN_AUTHORITY`,
    `AZURE_B2C_REDIRECT_URI` (falls back to `CLIENT_URL`),
    `AZURE_B2C_POST_LOGOUT_REDIRECT_URI` (falls back to `CLIENT_URL`)
- `angular.json` production config does `fileReplacements`: `environment.ts` → `environment.prod.ts`,
  and enforces budgets (`initial`: warn 1mb / error 2mb; `anyComponentStyle`: warn 10kb / error 20kb).

## Interfaces & contracts

This is a deployable SPA, not a published library — no package exports. Its external contract is
the HTTP API it consumes via `API_URL` (the gateway at `:8010`) and the Azure AD B2C OIDC endpoints.
Routing is module-based (`app-routing.module.ts` + per-feature `*-routing.module.ts`).

## Data & state

State is held in Angular **services** (e.g. `store.service.ts`, `basket.service.ts`,
`checkout.service.ts`, `core/services/loading.service.ts`) — there is **no NgRx** or external store
library. HTTP concerns flow through `core/interceptors/` (`loading.interceptor.ts`,
`error.interceptor.ts`).

## Dependencies

- Talks to the backend **API gateway at `:8010`** via `API_URL`.
- Authenticates against **Azure AD B2C** (MSAL + oidc-client).
- Standalone codebase — does **not** import the `@ecommerce-platform/*` packages used by
  `micro-frontends/`.

## Patterns

- Feature-module organization under `src/app/` (home/store/basket/checkout/account + core/shared);
  established by the existing module layout (`*.module.ts`, `*-routing.module.ts`).
- Environment config is generated at build time via `generate-env.cjs` — change the generator or its
  inputs, never hand-edit the generated `environment.prod.ts`.

## Gotchas

- This app is **legacy**; prefer new UI work in `micro-frontends/`. Confirm before investing here.
- **Linting is intentionally disabled** — `lint`/`lint:fix` just `echo ... && exit 0` in
  `package.json`. Don't rely on `npm run lint` to catch issues, even though `angular.json` still
  defines a real `lint` architect target.
- Production size budgets in `angular.json` are enforced; large additions can fail `build:prod`.
- `npm run build` defaults to the **production** configuration (`defaultConfiguration: production`),
  so a plain build applies budgets and prod file replacements.
