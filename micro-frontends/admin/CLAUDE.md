# admin — administration micro-frontend

## Do

- Use `adminThemeConfig` from `src/config/theme.ts` for Ant Design theming — this is purposely different from the `sharedThemeConfig` used by other remotes.
- Use the `inject(elementId, props)` / `unmount(elementId)` API from `src/injector.ts` as the host-facing contract.
- Thread `onLogout` from `props.config` through `RouterContext` so nested route components can trigger logout.
- Map `appContext` → `HostAuthContext` in `App.tsx` as shown — admin uses `HostAuthContext` directly, not the `useAuth()` hook pattern.
- Run `npm run routes:generate` (from `micro-frontends/`) after adding or renaming any file in `src/routes/` — `routeTree.gen.ts` is generated.
- Serialize URL search params via `createSearchSerializer(['group', 'service', 'configKey', 'env', 'tab'])`.

## Don't

- Do not import or use `sharedThemeConfig` from `@ecommerce-platform/shared-layout` here — admin has its own `adminThemeConfig`.
- Do not edit `src/routeTree.gen.ts` — it is auto-generated by `@tanstack/router-cli`.
- Do not add a new MSAL/`EcommerceAuthProvider` instance inside this remote; auth flows in from the host.
- Do not hardcode `basePath`; derive it from `appContext.basePath` at runtime.
- Do not pass arbitrary additional search params through the URL without registering them in `createSearchSerializer`.
