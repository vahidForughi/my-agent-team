# account — user account micro-frontend

## Do

- Use the `inject(elementId, props)` / `unmount(elementId)` API from `src/injector.ts` as the host-facing contract for this remote.
- Consume the host's auth context via `AuthConsumerProvider` and `useAuth()` from `@ecommerce-platform/auth-provider` — the host provides `user`, `token`, and `isAuthenticated` via `appContext`.
- Derive `basePath` from `appContext.basePath` passed in `AppInjectorProps`; read it in `InnerApp` via `RouterContext`.
- Run route generation (`npm run routes:generate` from the `micro-frontends/` root) after adding or renaming any file in `src/routes/` — `routeTree.gen.ts` is generated and must not be edited manually.
- Fetch user and order data exclusively through the TanStack Query hooks in `src/services/`.

## Don't

- Do not edit `src/routeTree.gen.ts` directly — it is generated by `@tanstack/router-cli`.
- Do not add MSAL or a new `EcommerceAuthProvider` instance inside this remote; auth comes from the host via `AuthConsumerProvider`.
- Do not hardcode `basePath` values; always read from `appContext.basePath`.
- Do not commit the `console.log` debug statements currently present in `src/App.tsx` to production builds.
- Do not serve this remote in isolation expecting full auth to work — standalone mode (`bootstrap.tsx`) does not include MSAL configuration.
