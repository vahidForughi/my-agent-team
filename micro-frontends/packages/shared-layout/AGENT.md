# Codebase Orientation Map

## 1-Line Summary
The `shared-layout` micro-frontend provides common UI components like Navbar, Footer, and a Layout wrapper for the ecommerce platform.

## 5-Minute Explanation
- **Primary tasks in code**: Provides reusable React components for overall application layout and navigation, handling both standalone and micro-frontend modes.
- **Primary inputs**: React props for component customization, browser routing for navigation, and authentication context from `@ecommerce-platform/auth-provider`.
- **Primary outputs**: Rendered UI components (Navbar, Footer, Layout) that are integrated into other micro-frontends.
- **Key files**:
    - `src/index.ts`: Entry point for the shared layout module.
    - `src/components/Layout/index.tsx`: Main layout component integrating Navbar and Footer.
    - `src/components/Navbar/index.tsx`: Navigation bar component.
    - `src/components/Footer/index.tsx`: Footer component.
    - `src/utils/navigation.ts`: Utility for handling navigation logic.
- **Main code paths**:
    1. Application starts, loads `shared-layout` via Module Federation.
    2. `src/index.ts` exposes components.
    3. Host application uses `<Layout>`, which then renders `<Navbar>` and `<Footer>`.
    4. Navigation events are handled by `src/utils/navigation.ts`.

## Deep Dive
- **Type**: Library / Micro-frontend component library
- **Primary runtime(s)**: Browser (React)
- **Entry points**:
  - `src/index.ts`: Exports `Layout`, `Navbar`, `Footer` components and utility functions.
  - `package.json`: Defines build scripts (`tsup`), dependencies (React, Ant Design, `@tanstack/react-query`, `react-router-dom`), and module exports.
  - `tsconfig.json`: Configures TypeScript compilation for the project.

## Top-Level Structure
| Path | Purpose | Notes |
|------|---------|-------|
| `src/` | Source code for shared layout components and utilities | Main implementation |
| `src/components/` | Individual UI components like Navbar, Footer, Layout | Organized by component |
| `src/constants/` | Constants and configuration related to the UI | e.g., theme, navigation items |
| `src/utils/` | Utility functions, especially for navigation | Shared helper logic |
| `dist/` | Compiled output of the TypeScript code | Generated by `tsup` |

## Key Boundaries
- **Presentation**: `src/components/` (all components)
- **Application/Domain**: `src/utils/navigation.ts` (navigation logic), components handle their own state/props
- **Persistence/External I/O**: None directly; relies on host application for data fetching (e.g., via `@tanstack/react-query` peer dependency).
- **Cross-cutting concerns**: Authentication via `@ecommerce-platform/auth-provider` (peer dependency), Ant Design for UI framework.
- **Responsibilities by file/module**:
    - `src/components/Layout/`: Orchestrates the overall page structure.
    - `src/components/Navbar/`: Handles top-level navigation and user actions.
    - `src/components/Footer/`: Displays bottom-of-page information.
    - `src/utils/navigation.ts`: Manages routing and navigation behaviors.
    - `src/constants/theme.ts`: Defines shared theming configurations.
- **Detailed code flows**:
  1. A micro-frontend application imports components from `shared-layout`.
  2. For example, `<Layout>` is used to wrap page content.
  3. Inside `<Layout>`, `<Navbar>` and `<Footer>` are rendered.
  4. `Navbar` uses `react-router-dom` for navigation and potentially `navigation-handler.tsx` for dynamic routing.
  5. `Auth` context is provided by the host application or `@ecommerce-platform/auth-provider`.
  6. Components interact with each other via props.
- **How the pieces map together**: Components are organized in `src/components`, utilities in `src/utils`, and constants in `src/constants`. `index.ts` serves as the public API. `package.json` defines the build process and dependencies.
