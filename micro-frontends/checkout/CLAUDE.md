# checkout — checkout flow micro-frontend

## Do

- Call `setupMocks()` before mounting the app (as done in `src/injector.ts`) to initialize MSW mock handlers.
- Use `inject(elementId, props)` / `unmount(elementId)` from `src/injector.ts` as the host-facing contract.
- Consume auth from the host via `AuthConsumerProvider` and `useAuth()` — checkout requires authentication; the host's `ProtectedRoute` enforces this.
- Derive `basePath` from `appContext.basePath` passed through `AppInjectorProps`.
- Run `npm run routes:generate` (from `micro-frontends/`) after adding or renaming any file in `src/routes/`.
- Fetch basket and order data through TanStack Query hooks in `src/services/basket/`.

## Don't

- Do not remove the `setupMocks()` call in `src/injector.ts` without understanding its production-vs-dev implications.
- Do not edit `src/routeTree.gen.ts` — it is auto-generated by `@tanstack/router-cli`.
- Do not add a new `EcommerceAuthProvider` or MSAL instance; auth flows in from the host.
- Do not try to load checkout without authentication — it is a protected route in the host shell.
- Do not hardcode `basePath`; always read it from `appContext.basePath`.
