# Workspace — big picture

The big-picture map of this repository for the harness. Fill this in during onboarding.

Each **part** of the workspace gets its own flat directory here named `_<part>/`
(e.g. `_services/`, `_client/`, `_api-gateways/`, `_micro-frontends/`, `_infrastructure/`,
`_deployments/`, `_monitoring/`). There is **no `parts/` wrapper and no deeper nesting** —
part dirs live directly under `workspace/`.

Every part dir contains:

- `README.md` — what the part is, where it lives in the repo, how to run/test it.
- `AGENT.md` — agent guidance and reusable patterns/gotchas for that part.
- `suggest/` — a place to drop suggestions, ideas, and follow-ups for that part.

See `_[name-part]/` for the copyable template.
